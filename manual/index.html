Physics webR Lab 学習ガイド

このページは、実験で測ったデータを表に入れ、R のコードでグラフを描いて読み解くまでを、順を追って学ぶための生徒向けガイドです。

ヒント前提にしていること
  • Google Colab などで Python を少し触ったことがある（変数・代入・関数呼び出しが何となく分かる）
  • ただし 「データフレーム」や「コードでグラフを描く」のは初めて

この 2 つを橋渡しするのがこのページの目的です。コピペして動かしながら読み進めてください。

0. このアプリで何をするのか

  1. 左の表に、実験で測った数値を入力する（1 行目は列の名前）。
  2. 右上のエディタに R のコードを書いて「コード実行」を押す。
  3. 計算結果は右下の コンソール、グラフは グラフ タブに出る。

データはブラウザの中だけで処理され、どこにも送信されません。まずは「設定読込 (JSON)」からサンプルを開いて、実物を見るのが近道です。

1. 「データフレーム」って何？

データフレーム (data.frame) は、ひとことで言うと 表 です。Excel やスプレッドシートの 1 枚のシートとほぼ同じで、

  • 列 (column) … 「長さ」「周期」など、1 種類の量のたて 1 列
  • 行 (row) … 1 回ぶんの測定（よこ 1 行）

でできています。Python で習った「リスト」が縦に何本も束ねられ、それぞれに名前が付いたもの、とイメージするとよいです。

このアプリでは、左の表がそのままデータフレームになります。1 行目に書いた言葉が列名になります。

2. まずはデータを受け取る — get.input()

左の表を R に取り込むには、get.input() を使います。

df <- get.input()   # 左の表を df という名前のデータフレームに入れる
head(df)            # 先頭の数行を表示して中身を確認する
  • <- は Python の =（代入）と同じだと思ってください。R では <- をよく使います。
  • head(df) は最初の数行だけを見せてくれます。summary(df) を使うと、列ごとの最小・最大・平均などの要約が出ます。
ノート列を 1 本だけ取り出す

df$length_m のように データフレーム$列名 と書くと、その列（縦 1 本）だけを取り出せます。あとでグラフの横軸・縦軸に使います。

ヒントやってみよう

「単振り子」サンプルを開き、df <- get.input() のあとに head(df) と summary(df) を実行して、表とまったく同じ数字が出ることを確かめましょう。

3. まずグラフを 1 枚描く — plot()

いちばん基本の作図は plot(横軸, 縦軸) です。

df <- get.input()
plot(df$length_m, df$period_1_s)   # 横軸 = 長さ, 縦軸 = 1回目の周期

これだけで散布図が グラフ タブに出ます。plot() に渡しているのは「横軸の列」と「縦軸の列」の 2 本です。

4. グラフの見た目を整える

plot() には、見た目を決めるオプションを足せます。よく使うものだけ覚えれば十分です。

set.paper_style()                  # 余白や軸を、レポート向けに整える(このアプリ独自)
plot(df$length_m, df$period_1_s,
  pch = 19,                        # 点の形(19 = 塗りつぶしの丸)
  col = "firebrick",               # 点の色
  cex = 1.2,                       # 点の大きさ
  xlab = "Length / m",             # 横軸ラベル(単位もここに書く)
  ylab = "Period / s",             # 縦軸ラベル
  main = "Pendulum"                # グラフのタイトル
)
add.paper_guides()                 # 補助のグリッド線を足す(このアプリ独自)
重要ラベルは英語で

このアプリの R はブラウザの中で動くため、グラフの中の日本語は文字化けします。xlab ylab main などは英語（Length / m など）で書いてください。コードの コメント（# のうしろ）は日本語でかまいません。

軸ラベルには 必ず単位 を書くこと、タイトルを大きくしすぎないこと、点や線は印刷しても見える太さにすること。これだけで「読めるグラフ」になります。

5. 新しい列を計算して足す — 平均など

繰り返し測定したら、平均 を求めて 1 本の列として足すのが基本です。「データフレーム$新しい列名 <- 計算 」で列を追加できます。

# 3回(または5回)測った周期の列をまとめて指定
period_cols <- c("period_1_s", "period_2_s", "period_3_s")

# 行ごとの平均を、新しい列 period_mean_s として足す
df$period_mean_s <- rowMeans(df[period_cols])
  • c(...) は 値をいくつか束ねる関数（Python のリストに近い）。ここでは列名を束ねています。
  • rowMeans() は 行ごと（測定ごと）の平均を計算します。

6. 欠損値 (NA) のあつかい

実験では「測り忘れ」「失敗して空欄」がよくあります。R では空欄は NA（欠損値） として扱われます。NA をそのまま計算すると、結果も NA になってしまいます。

対処は 2 つ覚えれば十分です。

# (A) 計算するとき、NA を無視する: na.rm = TRUE を付ける
df$period_mean_s <- rowMeans(df[period_cols], na.rm = TRUE)

# (B) 使えない行ごと取り除く: subset() で条件に合う行だけ残す
df <- subset(df, !is.na(length_m))   # length_m が NA でない行だけ残す
  • is.na(x) は「x が NA かどうか」を調べます。! は「ではない」の意味なので、!is.na(length_m) は「length_m が NA でない」。
  • 横軸に使う列が欠けている行は計算できないので、(B) で取り除きます。測定回数の一部だけ欠けているなら、(A) で「ある分だけで平均」します。

7. ばらつきとエラーバー

平均だけでなく、どれくらいばらついたかも大事です。ばらつきは 標準偏差 (sd) で表し、グラフでは 誤差棒（エラーバー） として描きます。

# 行ごとの標準偏差を新しい列にする(これが誤差棒の長さになる)
df$period_sd_s <- apply(df[period_cols], 1, sd, na.rm = TRUE)

apply(表, 1, sd) は「各行 (1) に対して sd を計算する」という意味です（2 にすると列ごと）。

誤差棒は arrows() で、平均から上下に sd ぶん伸ばして描きます。

plot(df$length_m, df$period_mean_s, pch = 19,
  xlab = "Length / m", ylab = "Period / s")

arrows(
  df$length_m, df$period_mean_s - df$period_sd_s,   # 下端 = 平均 - sd
  df$length_m, df$period_mean_s + df$period_sd_s,   # 上端 = 平均 + sd
  angle = 90, code = 3, length = 0.04               # 両端に短い横棒(矢じり)を付ける
)
ノートエラーバーの「量」を変える

± sd（標準偏差）が基本ですが、平均値のばらつきを示したいときは 標準誤差 sd / sqrt(n) を使うこともあります。df$period_sd_s の代わりに、伸ばす量の列を差し替えるだけで変えられます。

8. 直線・曲線をあてはめる（回帰）

データに直線をあてはめるには lm()（線形モデル）を使います。

# period_mean_s を length_m で説明する直線をあてはめる
fit <- lm(period_mean_s ~ length_m, data = df)

abline(fit, col = "#374151", lwd = 2)   # あてはめた直線をグラフに重ねる
summary(fit)                            # 傾き・切片・あてはまり具合をコンソールで見る
  • y ~ x は「y を x で説明する」という R の書き方です。
  • 曲線（例：2 次関数）をあてはめたいときは lm(y ~ x + I(x^2), data = df) のようにします。曲線は abline では描けないので、predict() で計算した値を lines() でつなぎます（→ スニペット集を参照）。

9. 3 つのタブの使い分け

タブ 何が見えるか
コンソール head() summary() print() の文字の結果、エラーメッセージ
グラフ plot() などで描いた図。「全体表示」で拡大、PNG 保存も可能
データフレーム R の中にある data.frame の一覧と先頭プレビュー（列が増えたか確認に便利）

10. 保存と読込

  • 設定保存 (JSON) … 表・スクリプト・コンソール履歴をまとめて 1 ファイルに保存。
  • 設定読込 (JSON) … 保存したファイル、または授業用サンプルを開く。
  • 読み込んだ後は、必要に応じて「コード実行」をもう一度押してください。

11. グラフのサイズ・縦横比・画面分割

ふだんはグラフが描画エリアの大きさに自動で合わせて描かれますが、コードで縦横比や余白、画面分割を自分で決めることもできます。

縦横比をそろえる（データの単位を 1:1 に）

plot() に asp = 1 を付けると、x と y の 1 目盛りが同じ長さになります。位置(x, y)のように両軸が同じ単位のときは、これで本当の形が崩れずに見えます。

plot(df$x_m, df$y_m, pch = 19, asp = 1,
  xlab = "x / m", ylab = "y / m")

図全体のサイズ・縦横比を決める — set.figure()

set.figure(width, height) で、図そのものの大きさと縦横比を指定できます。単位は units = "in"（インチ・既定）/ "cm" / "px"。

set.figure(width = 4.5, height = 4.5, units = "in")  # 正方形の図にする
plot(df$x_m, df$y_m, pch = 19, asp = 1,
  xlab = "x / m", ylab = "y / m")

set.figure() を呼ばなければ、これまで通り描画エリアに自動でフィットします。元に戻したいときは reset.figure() を実行します。

ノート余白(マージン)を変える

set.paper_style(mar = c(下, 左, 上, 右)) で、軸ラベルや目盛りのための余白を調整できます。ラベルが切れるときは左や下を大きく、余白が広すぎるときは小さくします。

画面を分割して並べる（subplot）— par(mfrow)

par(mfrow = c(行, 列)) を実行してから複数の plot() を呼ぶと、1 枚の図に複数のグラフを並べて描けます。

set.figure(width = 8, height = 3.5, units = "in")  # 横長にする
par(mfrow = c(1, 2))                               # 1 行 2 列に分割
plot(df$x_m, df$y_m, pch = 19, main = "x-y")       # 左
plot(df$t_s, df$y_m, type = "o", main = "y-t")     # 右
ヒントやってみよう

「作図サイズとレイアウト」サンプルを開いて実行し、set.figure() の数値や par(mfrow) の行・列を変えて、図の形がどう変わるか試してみましょう。

PNG 保存や「全体表示」も、指定した縦横比のまま書き出されます。
