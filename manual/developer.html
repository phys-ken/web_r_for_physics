開発者向けリファレンス

Physics webR Lab の内部構造リファレンス。LLM エージェント・開発者向けに、改変と再ホスティングに必要な事実のみを記す。

リポジトリ: https://github.com/phys-ken/web_r_for_physics ／ 公開アプリ: https://phys-ken.github.io/web_r_for_physics/ ／ ライセンス: MIT

全体像

  • 単一ファイルアプリ。アプリ本体のロジックは index.html（約 4400 行）に集約。末尾の単一 <script type="module"> ブロックが全 JS。
  • ビルド不要。GitHub Pages が index.html をそのまま配信（.nojekyll あり）。バンドラ・トランスパイルなし。
  • webR は CDN から ESM import: https://webr.r-wasm.org/v0.6.0/webr.mjs。v0.6.0 固定（latest の破壊的変更回避）。

状態モデル

  • state（index.html 内）が単一の信頼できる情報源。主な部分木:
    • state.app.workbook.sheets[] — シートのグリッド（行優先の二次元文字列配列、行 0 = ヘッダ）
    • state.app.script — エディタ内容
    • state.app.console.{entries, history} — コンソール出力と REPL 履歴
    • state.app.ui — currentPage / outputTab / status など
    • state.app.dataframes — data.frame 一覧と選択中プレビュー
    • state.webR — 起動済み WebR インスタンス（boot() で設定）
    • state.ready / state.busy — 実行ガード
  • elements — 全 getElementById 参照をまとめたオブジェクト。

R 実行フロー

  1. runRCode() — 中心の入口。データ同期＋実行を行う。
  2. syncWorkbookToR() → buildWorkbookRLiteral() → sheetToFrameModel()。各シートのグリッドを型推論（inferColumnType）して R の型付きベクタへ変換し、.GlobalEnv に注入する。
  3. executeRSource() が shelter.captureR() 内で source(..., print.eval = TRUE, local = .GlobalEnv) 包みでコードを評価。トップレベル式の出力もこれで拾う。
  4. グラフは captureR() の captureGraphics で取得。webR.stream() のストリーミング canvas パッチは意図的に無視（handleCanvasMessage 参照。複雑な増分描画で不整合になるため、最終ビットマップのみ採用）。
  5. 実行後 refreshDataFrameState() が .GlobalEnv を走査して DataFrame タブを更新。

get.input() と注入されるヘルパー

boot() 内で R 側に定義される関数（index.html の R リテラル中）:

  • get.input(sheet = NULL) — 現在シート / 名前 / 1 始まり番号でシートを data.frame として返す。実体は注入済みの .webr_input_sheets（名前付きリスト）と .webr_active_sheet_index を参照。
  • set.paper_style(mar, mgp, cex, tcl, las) — par() をレポート向け体裁に設定。
  • add.paper_guides(show_grid, show_top_axis, show_right_axis, grid_col) — グリッドと補助目盛り・box() を追加。
  • set.figure(width, height, units = "in"|"cm"|"px") / reset.figure() — .webr_fig_width_px / .webr_fig_height_px（既定 NA）に px 換算値を書き込む。クランプは 216–1600 px。

列の型推論は JS 側（inferColumnType）で行い、numeric / logical / character を判定してから R へ渡す。

図サイズ制御とレンダリング

  • captureR の captureGraphics: {width, height} は デバイス生成時にサイズを固定する。実測では出力ビットマップは指定 px の 2 倍解像度、アスペクト比は指定どおり。captureGraphics: true は常に 504×504、コード途中の options(device) 変更は無視される。
  • そのためサイズは キャプチャ前に JS が知る必要がある。resolveCaptureSize() はコードに set.figure / reset.figure が含まれるときだけ、スクリプトを サンドボックス事前評価（sink で出力抑制・pdf(NULL)・local = TRUE）してから .webr_fig_*_px を読み、captureGraphics のサイズに使う。未使用時はパネル実寸（state.plotSize）。
  • subplot は par(mfrow=...)、データ単位 1:1 は asp = 1 で、いずれも標準 base R のまま機能する。

複数画像ギャラリー

  • 1 回の実行で生成された 全プロットページ を扱う。executeRSource は capture.images 全体を renderCapturedImages() に渡す（以前は最後の 1 枚のみ）。
  • 各画像はネイティブ解像度の <canvas> として state.graphImages[] に保持し、state.graphSelectedIndex が選択中を指す。selectedGraphCanvas() がその 1 枚を返す。
  • renderGraphPreview() が選択画像をパネルへ contain（レターボックス） 描画。画像が 2 枚以上のとき renderGraphThumbs() が #graphThumbs にサムネイル（小 <canvas> + 「図 N」）を並べ、クリックで selectGraphImage()。
  • PNG 保存（downloadGraph、複数時はファイル名に -N）と全体表示（openGraphViewer、メタに「図 N / 総数」）はいずれも選択中の画像を対象とする。画像はエクスポート JSON には含めない（揮発的）。

JSON 保存・復元フォーマット

  • バージョン 2（payload.version === 2）。
  • 検証 validateAppPayload()、正規化 normaliseImportedPayload() / ensureWorkbookShape()。
  • UI 状態・コンソール履歴・ワークブック・スクリプトを丸ごと含む。サンプル（samples/*.json）も同一フォーマット。

入出力（Excel / CSV / xlsx / R / zip）

右上の 「入力」「出力」 ボタン（#inputButton / #outputButton）が入口。

  • 遅延依存: ensureSheetJS() が SheetJS を CDN から 動的 import（SHEETJS_URL、1 回だけ解決してキャッシュ）。初回 boot には影響させず、入出力操作の直前にのみ読み込む。WebR と同様 CDN 依存のためオフライン完全動作は不可。
  • 取り込み: handleDataFileSelected() → readDataFileToWorkbookModel()（csv/tsv は XLSX.read(text)、xlsx は array）。各シートを sheet_to_json(header:1) で二次元配列化し trimGrid()（末尾空行・空列の除去、IMPORT_MAX_ROWS/IMPORT_MAX_COLS で上限）。確認ダイアログ #importPreviewModal（openImportPreview / buildPreviewSheetSection / updatePreviewRename）でシート選択・見出し行・列名サニタイズ（sanitizeHeaderList 再利用）・置換/追加を提示し、confirmImportPreview() → applyImportedSheets() が state.app.workbook を更新して syncWorkbookToR()。単一シートの csv/tsv はファイル名をシート名に採用。
  • 書き出し: #exportModal を buildExportModalBody() が動的構築。runExport(kind) が種別ごとに分岐：exportDataSheets / exportLatestDataFrame / exportDataPlusDataFrame / exportRScript / exportSetup(JSON) / exportBundleZip。
    • データ→aoa は sheetToExportAoa()（末尾空列をトリムし inferColumnType で型付け、numeric/logical を実値化）。
    • data.frame は readDataFrameForExport() が buildDataFramePreviewRCode(name, DATAFRAME_EXPORT_MAX_ROWS)（プレビューと共通の hex プロトコル、行上限のみ拡大）で全行取得→ dataFramePreviewToAoa()。
    • xlsx は XLSX.write({type:"array"})、csv は csvBytes()（詳細設定で UTF-8 BOM 付与可）。
    • R スクリプト出力は buildExportScriptText()：/\bget\.input\s*\(/ を含む行を # でコメントアウトし、先頭に注意書きを付ける。
    • 一括出力は依存ゼロの store-only zip（buildZip() + crc32()、UTF-8 ファイル名フラグ）に data.xlsx / dataframe-*.csv / script.R / setup.json / plot.png（selectedGraphCanvas を PNG 化）を同梱。
    • ファイル名は timestampLabel() で日時付与、downloadBlob() で保存。

サンプル

  • samples/manifest.json が { id, title, description, file } の配列を持つ。
  • 各サンプルは単体の version 2 エクスポート JSON。loadSampleById() が必要時に fetch。
  • 追加するには JSON を samples/ に置き、manifest に 1 エントリ追記するだけ。

マニュアル

  • manual-src/ が Quarto ソース（.qmd + _quarto.yml + styles.css）。
  • manual/ がリポジトリにコミットされる静的出力で、Pages から配信。
  • .qmd 編集後は npm run manual:render（quarto render manual-src → manual/ へコピー）で再生成。

コマンド

npm start              # http://127.0.0.1:4173/ で配信 (python3 -m http.server)
npm test               # 静的スモークテスト → Playwright
npm run test:static    # node scripts/smoke-test.mjs のみ
npm run test:browser   # Playwright のみ
npm run manual:render  # Quarto マニュアル再生成
npx playwright install chromium   # 初回のみ
npx playwright test --grep "editor execution"   # 単一テスト

テスト

  • tests/app.spec.mjs に Playwright 7 件（fullyParallel: false で逐次）。
    1. ナビ → webR 起動 → サンプル読込 → スクリプト実行 → グラフ描画 → 全体表示 → 「出力」→ 設定 (JSON) エクスポート/インポート往復 → manual ページ。
    2. 表への TSV ペースト → エディタ実行で複数 data.frame → DataFrame タブのプレビュー。
    3. set.figure による図のアスペクト比制御。
    4. ニュートンサンプルの複数シート → 図ギャラリー切替。
    5. R スクリプト出力で get.input() 行がコメントアウトされること。
    6. データ領域の CSV 出力（SheetJS 遅延読込を経由）。
    7. CSV 取り込みで確認ダイアログ → シート展開 → get.input() に反映。
  • PLAYWRIGHT_BASE_URL で対象 URL を上書き可（既定 http://127.0.0.1:4173）。
警告テストが依存する不変点

サンプル 単振り子 の存在とその script 内の set.paper_style、#manualPage 内の a[href="./manual/"] リンク、manual トップの h1「Physics webR Lab 学習ガイド」と「スニペット集」リンク。改変時は壊さないこと。
